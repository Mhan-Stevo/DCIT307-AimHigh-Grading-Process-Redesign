## 📌 Project Title

**IT-Enabled Hybrid Redesign of the Student Grading and Report Generation Process at Aim High School Complex**

---

## 🏫 Organisation Overview

**Organisation Name:** Aim High School Complex  
**Sector:** Education  
**Type:** Private Educational Institution  
**Core Service:** Academic instruction and student performance evaluation  

Aim High School Complex currently uses a decentralized Microsoft Excel-based system for managing student grading and report generation. While functional, the process has resulted in operational inefficiencies, high error rates, and delayed report delivery to parents.

---

## 🚨 Problem Statement

The current Excel-based grading and report generation process at Aim High School Complex is characterized by:

- File duplication and multiple sheets for the same student
- Version conflicts across different users
- Manual calculation errors
- High data entry error rate (~20% reported)
- Slow consolidation of results across classes
- Lack of audit trail and change tracking
- Risk of data loss
- No real-time analytics or dashboards
- Manual student ranking process
- Manual, one-by-one report printing
- Average report preparation lead time of approximately 5 days after exams

These issues result in delayed report distribution, administrative stress, reduced data integrity, and inefficiencies in academic decision-making.

---

## 🎯 Project Objective

To analyze, document, and redesign the student grading and report generation process at Aim High School Complex using a hybrid IT-enabled approach that:

- Reduces report preparation lead time
- Minimizes data entry errors
- Eliminates version conflicts
- Improves auditability and data integrity
- Enables real-time performance analytics
- Automates ranking and batch report generation

---

## 📊 Measurable Success Metrics

The redesigned process aims to achieve:

- Reduce report release time from ~5 days to ≤ 2 days
- Reduce data entry error rate from ~70% to ≤ 10%
- Eliminate duplicate student sheets
- Remove version conflicts through centralized control
- Enable automated student ranking
- Enable batch report generation
- Provide dashboard analytics for management

---

## 🔎 Project Scope

### In Scope:
- Teacher score submission process
- Data entry and validation process
- Consolidation of class results
- Ranking and report generation
- Printing and distribution workflow

### Out of Scope:
- Teaching methodology
- Admissions process
- Fee payment systems
- School management outside grading workflow

---

## 🧠 Proposed Solution Approach (Hybrid Model)

The proposed solution will adopt a **Hybrid IT Model**, combining:

- Structured Excel-based data entry templates (controlled format)
- Centralized database backend
- Automated validation mechanisms
- Role-based access control
- Automated ranking and calculations
- Dashboard analytics for administrators
- Batch report generation capability
- Basic audit logging functionality

This approach balances cost, feasibility, usability, and scalability.
