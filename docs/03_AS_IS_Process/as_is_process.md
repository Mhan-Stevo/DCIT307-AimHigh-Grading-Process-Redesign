# Detailed Process Steps

### Step 1: Recording Student Scores
Teachers enter scores for tests, assignments, and examinations into Excel spreadsheets.

**Issues Observed**
- Data entry mistakes
- Incorrect formula usage
- Inconsistent formatting

### Step 2: Saving Individual Class Sheets
Each subject teacher saves grading sheets separately.

**Issues Observed**
- File duplication
- Multiple versions of the same sheet

### Step 3: Sharing Files with Administrative Staff
Teachers submit their grading sheets to administrative staff responsible for compiling results.

**Issues Observed**
- Files sent through different channels
- Missing or outdated files

### Step 4: Consolidation of Results
Administrative staff combine multiple Excel sheets into a single master file.

**Issues Observed**
- Very time-consuming
- High risk of copying errors
- Difficult to track updates

### Step 5: Calculating Totals and Averages
Totals and averages are calculated using formulas or manual calculations.

**Issues Observed**
- Formula errors
- Incorrect data references
- Inconsistent calculations

### Step 6: Student Ranking
Students are ranked within their class based on total scores.

**Issues Observed**
- Ranking done manually
- High chance of ranking errors

### Step 7: Generating Report Sheets
Administrative staff prepare report sheets for each student.

**Issues Observed**
- Repetitive manual formatting
- Slow generation process

### Step 8: Printing Reports
Reports are printed individually by printing staff.

**Issues Observed**
- Printing takes several hours
- Staff sometimes work overnight

### Step 9: Distribution to Parents
Final reports are distributed to students and parents.

**Issues Observed**
- Delayed report availability
- Parents receive reports several days after exams

---

# 5. Process Bottlenecks

Several bottlenecks were identified in the current grading system:

- **Manual Data Entry**: Teachers manually enter student scores, which leads to a high error rate.
- **File Version Conflicts**: Multiple versions of Excel files create confusion during consolidation.
- **Slow Data Consolidation**: Combining grading sheets across multiple subjects takes significant time.
- **Manual Ranking**: Ranking students manually increases workload and introduces errors.
- **Printing Delays**: The report printing process is slow and requires extensive manual effort.

# 8. Summary of AS-IS Process

The current grading system at Aim High School Complex is heavily dependent on manual effort and decentralized spreadsheet management.

While Excel provides basic computational capabilities, the lack of centralized data management results in:

- Data inconsistencies
- High error rates
- Slow report generation
- Increased administrative workload

These issues highlight the need for a redesigned grading system that introduces automation, centralized data management, and improved workflow coordination.
