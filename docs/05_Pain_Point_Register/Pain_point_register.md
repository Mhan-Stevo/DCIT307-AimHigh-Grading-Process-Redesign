# Pain Point Register
## Aim High School Complex – Grading Process

---

# 1. Introduction

The Pain Point Register identifies the major operational problems experienced in the current grading system.

These pain points were gathered through interviews with:

- Teachers
- Administrative staff
- IT department
- Printing personnel

---

# 2. Pain Point Register Table

| ID | Pain Point | Description | Impact | Severity |
|----|------------|-------------|--------|---------|
| PP1 | File Duplication | Multiple versions of grading sheets created | Conflicting data | High |
| PP2 | Version Conflicts | Different teachers submit different sheet versions | Data inconsistency | High |
| PP3 | Manual Calculation Errors | Incorrect formulas or references | Wrong results | High |
| PP4 | Slow Consolidation | Combining subject files manually | Delayed reports | High |
| PP5 | Lack of Audit Trail | No tracking of changes made to sheets | Accountability issues | Medium |
| PP6 | Data Loss Risk | Files sometimes deleted or overwritten | Loss of student records | High |
| PP7 | No Real-time Analytics | No system for instant performance analysis | Limited insights | Medium |
| PP8 | Manual Ranking | Students ranked manually | Ranking mistakes | Medium |
| PP9 | Printing Bottleneck | Reports printed individually | Long printing times | Medium |

---

# 3. Root Causes

The primary causes of these pain points include:

- Heavy reliance on Excel spreadsheets
- Lack of centralized data storage
- Limited automation
- Inconsistent file management practices

---

# 4. Summary

The pain points identified reveal serious inefficiencies in the current grading process.

Addressing these issues will require:

- Centralized data systems
- Automation of calculations
- Automated report generation
- Improved workflow coordination
